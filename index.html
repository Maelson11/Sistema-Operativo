Sistemas Operativos

os sistemas operacionais são programas ou softwares que gerenciam os recursos de hardware e permitem que outros programas funcionem em um computador. Eles atuam como intermediários entre o hardware e os programas de aplicação. Controlam tarefas como a gestão de memória, a administração de arquivos, a interação com dispositivos periféricos e facilitam a comunicação entre o hardware e o software. Além disso, proporcionam uma interface para que os usuários interajam com o computador.

Tema1:Exigências do sistema operativo quanto às capacidades do equipamento

As exigências do sistema operacional em relação às capacidades do equipamento se referem aos requisitos mínimos que um sistema precisa para funcionar de maneira adequada. Isso inclui especificações de hardware, como processador, memória RAM, espaço de armazenamento e outros recursos. Por exemplo, um sistema operacional pode demandar um processador de pelo menos 1 GHz, 2 GB de RAM e 20 GB de espaço em disco para ser instalado e funcionar corretamente. Esses requisitos podem variar de acordo com o sistema operacional específico e suas versões. Se um computador não atender aos requisitos mínimos do sistema operacional, é provável que ele funcione de maneira lenta, tenha problemas de desempenho ou, em alguns casos, não consiga executar o sistema operacional. É sempre recomendável verificar os requisitos do sistema antes de instalar um novo sistema operacional para garantir uma experiência adequada de uso.

Tema2:Instalação do SO a partir de periféricos

A instalação de um sistema operacional a partir de periféricos refere-se ao processo de instalar um sistema operacional em um computador utilizando dispositivos externos, como unidades de DVD, unidades USB ou até mesmo pela rede. Geralmente, esse processo envolve criar um meio de instalação inicializável, como um DVD ou uma unidade flash USB, que contenha os arquivos de instalação do sistema operacional. Em seguida, o computador é configurado para inicializar a partir desse dispositivo externo.

Os passos comuns para instalar um sistema operacional a partir de periféricos incluem:

  • Preparação do meio de instalação: Criar um DVD ou unidade USB inicializável com os arquivos de instalação do sistema operacional desejado.

  • Configuração da ordem de inicialização: Acessar a BIOS ou UEFI do computador para definir a ordem de inicialização, priorizando o dispositivo (DVD ou USB) onde está o arquivo de instalação do SO.

  • Inicialização a partir do periférico: Reiniciar o computador e permitir que ele inicie a partir do dispositivo escolhido. O sistema operacional carregará os arquivos de instalação.

  • Seguir as instruções de instalação: O processo de instalação do sistema operacional será iniciado a partir do periférico selecionado. Siga as instruções na tela para escolher o idioma, particionamento de disco, configurações e outras opções necessárias.

  • Conclusão da instalação: Após seguir todas as etapas de configuração e instalação, o sistema operacional será instalado no disco rígido ou na unidade de armazenamento do computador.

    • Tema3:Selecção de opções na instalação

    Na instalação de um sistema operacional, geralmente você encontra várias opções que podem ser configuradas para personalizar a instalação de acordo com suas preferências e necessidades. Essas opções variam de acordo com o sistema operacional específico.

    Mas aqui estão algumas opções comuns que você pode encontrar durante o processo de instalação:

  • Idioma e localização: Escolha o idioma em que deseja instalar o sistema operacional e configure a região ou localização para ajustar as configurações de data, hora e formato de teclado.

  • Tipo de instalação: Em alguns casos, você pode escolher entre uma instalação personalizada ou uma instalação rápida/padrão. A instalação personalizada permite configurar opções avançadas de particionamento de disco, enquanto a instalação rápida segue configurações padrão.

  • Particionamento de disco: Se a instalação permitir, você poderá criar, excluir ou modificar partições no disco rígido. Isso inclui a criação de partições para o sistema operacional, dados e outros fins.

  • Configurações de rede: Durante a instalação, é possível configurar a conexão com a rede, seja via Wi-Fi ou conexão com fio, inserindo informações como nome da rede (SSID) e senha.

  • Seleção de recursos e componentes: Alguns sistemas operacionais permitem escolher os recursos ou componentes a serem instalados. Por exemplo, em um sistema operacional mais avançado, você pode selecionar determinados pacotes de software ou recursos específicos.

  • Conta de usuário e senha: Configuração de conta de usuário, nome do computador e senha associada à conta administrativa.

    • Tema4:Definição do plano de configuração de periféricos

    O plano de configuração de periféricos se refere ao conjunto de etapas e decisões tomadas para configurar e integrar dispositivos periféricos a um sistema, como um computador, para garantir seu funcionamento correto. Isso inclui qualquer dispositivo externo conectado ao computador, como impressoras, scanners, discos rígidos externos, monitores, teclados, entre outros.

    Esse plano pode abranger vários aspectos:

  • Conexões físicas: Estabelecer conexões físicas entre os periféricos e o computador, seja por portas USB, HDMI, VGA, Thunderbolt ou outros tipos de conexão.

  • Drivers e software: Instalar os drivers apropriados e o software necessário para permitir a comunicação entre o sistema operacional e os periféricos. Muitas vezes, os dispositivos vêm com CDs de instalação ou é possível baixar os drivers mais recentes do site do fabricante.

  • Configurações de sistema: Realizar configurações no sistema operacional para reconhecer e otimizar o desempenho dos periféricos. Isso pode envolver ajustes de resolução de tela, configurações de energia, configurações de áudio, entre outros, dependendo do dispositivo.

  • Testes e verificação: Após a configuração, é importante testar os periféricos para garantir que estejam funcionando corretamente. Isso pode incluir imprimir um documento de teste em uma impressora, verificar a funcionalidade de um scanner, testar a qualidade de áudio de alto-falantes, entre outros testes específicos para cada dispositivo.

  • Atualizações e manutenção: Manter os drivers e o software dos periféricos atualizados é essencial para garantir um desempenho contínuo e cor.

    • Tema5:Definição do modelo de logins

    O modelo de logins se refere à estrutura ou ao padrão utilizados para gerenciar e controlar o acesso de usuários a um sistema, seja um sistema operacional, um aplicativo ou uma plataforma online. Esse modelo define as políticas, os procedimentos e as tecnologias empregadas para autenticar usuários e conceder permissões apropriadas.

    Existem vários tipos de modelos de logins, sendo os principais:

  • Login Único (Single Sign-On - SSO): Este modelo permite que um usuário acesse múltiplos sistemas ou aplicativos com apenas uma autenticação. Isso simplifica o processo para os usuários, que não precisam inserir credenciais repetidamente.

  • Autenticação Multifatorial (Multi-Factor Authentication - MFA): Esse modelo utiliza mais de um método de autenticação para verificar a identidade do usuário. Geralmente, envolve algo que o usuário sabe (senha), algo que possui (dispositivo) e algo que é (impressão digital, reconhecimento facial).

  • Autenticação Biométrica: Baseada em características físicas únicas do usuário, como impressão digital, íris, reconhecimento facial ou voz.

  • Modelo de login baseado em funções (Role-Based Access Control - RBAC): Atribui acesso com base nas funções que os usuários desempenham dentro de uma organização. Por exemplo, um administrador terá acesso a recursos específicos que um funcionário comum não teria.

  • Políticas de senha: Definem os requisitos para criar e manter senhas seguras, como comprimento mínimo, uso de caracteres especiais e periodicidade de troca.

  • Sessões e gerenciamento de tokens: Define como as sessões de login são gerenciadas, incluindo o tempo limite para sessões ativas e a emissão e revogação de tokens de autenticação.
